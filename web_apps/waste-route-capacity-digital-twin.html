Waste Route & Capacity Digital Twin

This prototype treats the decommissioning waste route as the real production system: characterization → segregation → size reduction → packaging → assay → interim storage → transport/disposal. The unit is intentionally generic (“package-equivalent”) so you can map it to drum-equivalents, container-equivalents, or volume-normalized units without smuggling fake precision into the model.

Scenario controls

The defaults are tuned so the model sits near the interesting frontier: assay and transport are both close to binding, which is where managerial intuition usually improves fastest.

Stress test settings
The model is deliberately austere: a finite-buffer queueing line with one rework loop. That is a feature, not a defect. It makes it obvious when “more field work” is merely creating more WIP that the waste route cannot absorb.

Deterministic run

Queue trajectories

End-of-day queues by stage. When storage saturates, the assay queue typically becomes the visible symptom, but transport is often the hidden cause.

Utilization by stage

Average use of available capacity across the horizon. Near-100% utilization is not automatically good if it is achieved by growing queues.

Bottleneck attribution

Counts the days each stage had the highest queue pressure. This is a pressure index, not a metaphysical statement about “the” bottleneck.

Throughput and lead time

Averages conceal the pathology. Watch what happens to lead time when you tighten one stage but leave downstream routing unchanged.
